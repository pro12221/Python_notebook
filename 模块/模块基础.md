## python文件的两种用途
```python
# __name__ 是一个特殊的变量，用于表示当前模块的名称
# 当模块作为主程序运行时，__name__ 的值是 "__main__"。
# 当模块被导入到其他模块时，__name__ 的值是模块的名称（即文件名去掉 .py 后缀）

# 如果直接运行 example.py，Python 会把 __name__ 设为 "__main__"，然后执行 hello() 函数
def hello():
    print("Hello, World!")

if __name__ == "__main__":
    hello()

# 如果在其他 Python 文件中导入 spam.py
# 那么 spam.py 中的 __name__ 将是 "spam"，而 hello() 函数不会自动执行。
import spam
print(spam.__name__)
```
## 循环导入
```python
# 当Python解释器遇到循环导入时，通常会导致以下问题：
# AttributeError 或 NameError：
# 在其中一个模块尝试使用另一个模块中的类、函数或变量时，由于被导入的模块尚未完全加载（即代码块还没有执行完毕），目标对象可能尚未被定义或绑定到名称空间，从而引发错误。
# 模块部分加载：
# Python在导入模块时会执行其顶层代码。当发生循环导入时，其中一个模块在执行时会尝试导入另一个模块，而那个模块可能处于正在加载的状态。这会导致一个模块只能访问另一个模块已加载部分（即在循环发生点之前已经执行的代码）。

# 解决循环导入方法
# module_a.py
def some_function():
    # 延迟导入：只在函数被调用时才执行导入
    import module_b
    result = module_b.get_data()
    return result

# module_b.py (保持不变，顶层导入 module_a)
import module_a 
# ...
```
